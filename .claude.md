# log-context-mcp — Project Guide for Claude

**Purpose**: Token-efficient log preprocessing for LLM coding agents. An MCP server that analyzes and deduplicates log files to produce structured summaries in 500-2000 tokens instead of 15,000+.

## Project Structure

```
log_context_mcp/
├── __init__.py                 # Package metadata
├── server.py                   # MCP server & tool definitions (entry point)
├── preprocessor.py             # Layer 1: Deterministic analysis (free)
└── analyzer.py                 # Layer 2: Semantic analysis (LLM-powered, cheap)

tests/
├── __init__.py
└── test_log_context.py        # 56 tests covering all layers

pyproject.toml                   # Package config + pytest settings
README.md                        # User documentation
```

## Architecture

**Three-layer design**:
- **Layer 1 (preprocessor.py)**: Deterministic log processing — dedup, severity detection, stack trace grouping. Zero cost, ~50-70% compression.
- **Layer 2 (analyzer.py)**: Semantic analysis using LLM (currently Claude Haiku, but backend-agnostic). Classifies errors, extracts root cause, builds timeline.
- **Layer 3 (server.py)**: MCP tools for drill-down queries. Agent can request specific lines by pattern or severity.

## Backend-Agnostic LLM Support

As of the latest refactor, **analyzer.py** supports any LLM backend via OpenAI-compatible API or Anthropic native API.

### Auto-Detection Order
1. `LOG_CONTEXT_BACKEND` env var (explicit `anthropic`, `openai`, or `ollama`)
2. `ANTHROPIC_API_KEY` → Anthropic native API
3. `OPENAI_API_KEY` → OpenAI-compatible (covers Groq, Together, OpenAI, etc.)
4. Ollama at `http://localhost:11434` → auto-detected
5. None found → deterministic-only mode (Layer 1 only)

### Relevant Env Vars
- `ANTHROPIC_API_KEY` — Anthropic
- `OPENAI_API_KEY` — OpenAI or compatible provider
- `OPENAI_BASE_URL` — Override base URL (e.g., Ollama local endpoint)
- `LOG_CONTEXT_BACKEND` — Explicit backend override
- `LOG_CONTEXT_MODEL` — Override default model name per backend

### Default Models
- Anthropic: `claude-haiku-4-5-20251001`
- OpenAI: `gpt-4o-mini`
- Ollama: `llama3`

## Key Files & Entry Points

- **server.py**: `mcp.run()` — entry point for Claude Code integration
- **preprocessor.py**: `preprocess(raw_text: str) → PreprocessorResult` — deterministic Layer 1
- **analyzer.py**: `analyze(result: PreprocessorResult, api_key: Optional[str]) → Optional[SemanticAnalysis]` — Layer 2

## Testing

All tests are in `tests/test_log_context.py` (56 tests, ~72% coverage).

```bash
# Run all tests
python3.11 -m pytest tests/ -v

# With coverage
python3.11 -m pytest tests/ --cov=log_context_mcp --cov-report=term-missing

# Specific test class
python3.11 -m pytest tests/test_log_context.py::TestFullPreprocessing -v
```

**Test coverage**:
- Layer 1 (deterministic): 35 tests — ANSI stripping, severity, dedup, timestamps, noise, stack traces, full pipeline
- Prompt generation: 4 tests — without API calls
- Backend resolution: 13 tests — env var handling, auto-detection
- Backend implementations: 4 tests — API request formatting with mocks

**Key design**: Tests run **without external APIs** — all Layer 2 API calls are mocked.

## Import Conventions

After the recent refactor, all imports must be **fully qualified**:

```python
# ✓ CORRECT
from log_context_mcp.preprocessor import preprocess
from log_context_mcp.analyzer import analyze

# ✗ DO NOT USE (old style)
from preprocessor import preprocess
```

This applies to both external imports and internal package imports.

## Common Tasks

### Running the Server

```bash
# Install first
pip install -e .

# Register with Claude Code
claude mcp add log-context -- python /path/to/log-context-mcp/log_context_mcp/server.py

# Or run directly with a test
ANTHROPIC_API_KEY=sk-ant-... python3.11 -c "
import asyncio
from log_context_mcp.analyzer import analyze
from log_context_mcp.preprocessor import preprocess
# ... test code
"
```

### Adding a New Backend

1. Create a new backend class in `analyzer.py` similar to `_AnthropicBackend` or `_OpenAICompatibleBackend`
2. Add env var detection in `_resolve_backend()` function
3. Update default model constants
4. Add test cases in `tests/test_log_context.py`

### Testing a Single Modification

```bash
# Install dev deps
pip install -e ".[dev]"

# Run tests
python3.11 -m pytest tests/ -v -k "test_name_pattern"

# With coverage for just modified files
python3.11 -m pytest tests/ --cov=log_context_mcp.analyzer --cov-report=term-missing
```

## Code Style & Conventions

- **Type hints**: Required for all function signatures
- **Docstrings**: Use for public functions and classes (e.g., `analyze()`, backend classes)
- **Error handling**: Return `None` gracefully when external APIs fail — deterministic layer always works
- **No breaking changes**: `analyze()` signature must remain backward compatible
- **Logging**: Use stderr for non-critical errors (see existing error handling in `analyzer.py`)
